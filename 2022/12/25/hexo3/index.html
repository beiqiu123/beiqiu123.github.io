如何将hexo博客一键部署到云服务器

发表于2022-12-25|更新于2022-12-25|hexo基础教程
|字数总计:1k|阅读时长:3分钟|阅读量:

关于本地搭建环境配置、hexo博客搭建及如何上传至github本文不再赘述，请参考另外一篇文章Hexo博客搭建及Vercel部署

0. 前言

为了解决 hexo 部署到 GitHub 访问速度慢的问题，我们可以将 hexo 部署到云服务器上。
本文将介绍如何使用宝塔面板将 hexo 部署到阿里云服务器上，服务器系统选用的是Centos8.2。

1. 搭建环境

  • 成功搭建hexo并且部署至github
  • 一台云服务器（最好2C2G以上，本文使用的是2C4G服务器）
  • 完成备案的域名（非中国内地服务器可以不备案）
  • SSH终端以连接到远程服务器。（Xshell、宝塔远程工具、或者云服务器控制台自带的VNC都可以）

2. 宝塔面板安装

运用宝塔面板的原因是其可以可视化的远程操作你的服务器，避免手动部署环境的繁琐，但是如果处于安全性考虑请自行手动部署环境。这小博客谁来黑啊！
进入宝塔面板的下载页面宝塔面板下载 ，找到 Linux 面板，点击安装脚本，如果服务器系统是 Windows 的就点 Windows 的，但是 Windows 系统对于内存占用较高所以推荐使用 Linux 系统。

根据自己服务器的类型，复制对应的代码

打开你的终端连接至服务器，粘贴刚才复制的代码，回车，一路确认，然后在浏览器中访问外网面板地址，使用宝塔提供的用户名和密码登录。

进入面板后，点击一键安装 LNMP。就能直接将 Nginx，MySQL 等网站所需内容一键安装好，但是 hexo 部署实际只需要 Nginx ，所以可以只安装部署 Nginx 。

3. 安装并配置 Git 仓库

在远程服务器上配置好 Git 仓库后，才能将本地的 hexo push 到远端。

  1. 在云服务器上安装 git
  2. 配置 git 用户，赋予用户权限并且编辑相关文件
  3. 添加 git 用户登录
    输入 i 进入insert模式 ，找到root ALL=(ALL) ALL，在其下方加入一行git ALL=(ALL) ALL(输入的时候请删除 + 号) 按下ESC，输入 :wq ，保存 vim 文件。
  4. 修改 sudoers 文件权限
  5. 设置 git 用户的密码
    在Centos系统下输入密码是不会显示的，所以输入完直接按Enter。
  6. 给 git 用户添加 ssh 秘钥
    在部署 hexo 到 Github 时生成了一个秘钥，默认位置在C:\Users\用户名\.ssh\id_rsa.pub 。
  7. 在本地打开一个终端，以ssh的方式登录云服务器
  8. 创建一个git仓库，新建一个post-receive文件，用来存储本地的提交。 在blog.git/hooks文件夹下创建一个post-receive钩子，把提交到git仓库的文件同步到home/hexo文件夹中 在post-receive文件中输入以下代码
  9. 授予post-receive文件可执行权限 至此服务端已经配置完成。

    打开hexo的配置文件_config.yml修改Deployment位置的配置 保存配置文件，打开本地终端进入hexo博客所在文件夹 此时应该已经可以通过 ip:80 来访问了。
