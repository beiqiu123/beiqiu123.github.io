Hexo博客搭建及Vercel部署

发表于2022-12-12|更新于2022-12-13|hexo基础教程

1. 搭建环境

本教程搭建环境采用Win10作为操作系统，因此主要面向Windows用户

  • 操作系统：Windows10
  • Node
  • Git
  • Hexo
  • 文本编辑器(强烈推荐VSCODE)
  • GitHub 帐号（如登陆不上请自行使用魔法解决）
  • Vercel
  • 一个域名（强烈推荐买个域名）
  • 云服务器（可以使用Vercel替代，可选）

2. Node安装

  1. 打开Node官网，下载和自己系统相配的Node的安装程序，否则会出现安装问题。下载地址：https://nodejs.org/en/download/
    我个人的版本是 12.19.0，目前最新版本已经更新到19.0.0，建议选择相对低一些的版本，可以和我的一样，否则在搭建过程中容易出现各种不兼容的问题！
    历史版本下载：https://nodejs.org/en/download/releases/

    找到自己想要的版本后点击Downloads后会跳转至以下页面，选择对应的操作系统版本安装，这里选的的是win-x64
  2. 下载后安装，安装的目录可以使用默认目录【C:/Program Files/nodejs/】，也可以自定义路径。
    尽量选择默认路径，否则需要自行配置系统环境变量，方法请自行百度这里不再概述。
  3. 安装完成后，检查是否安装成功。在键盘按下win+R，输入CMD，然后回车，打开CMD窗口，执行node -v命令，看到版本信息，则说明安装成功。
    如果以上步骤完全正确且选择默认路径的情况下执行命令后无法查看到版本信息，那么请把相同版本号的Node的win-32版本也安装一下再次查看。

10. Vercel部署与自定义域名

10.1 Vercel的部署

Vercel简介：vercel是一个代码托管平台，它能够托管你的静态html界面，甚至能够托管你的node.js与Python服务端脚本，因此可以替代服务器的作用，最重要的是免费！！
使用Vercel部署Hexo项目步骤：

  1. 首先需要一个Vercel账号，推荐使用你刚刚注册的GitHub账户关联，这样就可以在vercel中直接找到并且托管你的GitHub库中的项目了（网络不流畅或是网页加载不出请使用魔法）。
  2. 当你用你的Github账户关联并绑定手机号登录之后，点击右上角的Add New Project创建新的项目，之后导入选项那里选择Continue with Github，这时候应该能看到你Github账号的仓库，选择你刚刚部署成功的存储静态博客的仓库< username >.github.io右边的Import选项，表示你要导入该仓库。
  3. 起一个只能有字母、数字或者或者连字符的项目名称，然后其他默认，点击Deploy，等待片刻即可部署成功，部署成功后点击Continue to Dashboard跳转到控制面板，如下图所示，但是我们现在还不能访问他给出的域名，因为该域名被墙了。

    10.2 绑定自定义域名

    现在你的个人网站的地址是username.github.io以及在Vercel上有一个地址blog-demo.vercel.app。如果觉得不够定制化，可以购买一个专属域名。

    这一步可选，如果目前不想买域名可以先跳过，但是强烈建议使用个人域名，阿里云、腾讯云、华为云等厂商注册域名一年二十块，也可以使用freenom注册免费域名，但是可能需要魔法且无法备案或是解析到国内服务器

    腾讯云、阿里云、华为云等都是不错的域名代理商，建议选用com，cn或cc这类常用好记的顶域，尽量不要使用汉字顶域，这样对SEO（即搜索引擎优化）比较友好便于收录，自定义域名尽可能短而好记，但是不要少于5个字符否则可能会有额外收费。
    以我的域名：yhmyhm.cn为例进行说明
主机记录 解释
www 解析后的域名为 www.yhmyhm.cn
@ 直接解析主域名 yhmyhm.cn
* 泛解析，匹配其他所有域名 *.yhmyhm.cn
mail 将域名解析为 mail.yhmyhm.cn，通常用于解析邮箱服务器
二级 如 abc.yhmyhm.cn，填写 abc
手机网站 如 m.yhmyhm.cn，填写 m
显性URL 不支持泛解析（泛解析：将所有子域名解析到同一地址）

将域名指向云服务器，请选择「A」； 将域名指向另一个域名，请选择「CNAME」； 建立邮箱请选择「MX」，根据邮箱服务商提供的 MX 记录填写。

记录类型 解释
A 用来指定域名的 IPv4 地址（如 8.8.8.8），如果需要将域名指向一个 IP 地址，就需要添加 A 记录。
CNAME 如果需要将域名指向另一个域名，再由另一个域名提供 IP 地址，就需要添加 CNAME 记录。
MX 如果需要设置邮箱，让邮箱能收到邮件，就需要添加 MX 记录。
TXT 在这里可以填写任何东西，长度限制 255。绝大多数的 TXT 记录是用来做 SPF 记录（反垃圾邮件）
NS 域名服务器记录，如果需要将子域名交给其他 DNS 服务商解析，就需要添加 NS 记录。
AAAA 用来指定主机名（或域名）对应的 IPv6 地址（例如：ff06:0:0:0:0:0:0:c3）记录。
SRV 记录了哪台计算机提供了哪个服务。格式为：服务的名字、点、协议的类型，例如：_xmpp-server_tcp。
显性URL 从一个地址 301 重定向到另一个地址的时候，就需要添加显性 URL 记录（注：DNSPod 目前只支持 301 重定向）。
隐性URL 类似于显性 URL，区别在于隐性 URL 不会改变地址栏的域名。

添加自定义域名的步骤：

  1. 点击Vercel控制面板右上角的View Domains查看当前的域，我们可以看到仅有Vercel给你预分配的一个域名，此时我们输入刚刚购买的域名，我这里以我的域名·为例进行说明，添加后他会提示你添加一条DNS解析记录。

    添加一级域名后他会推荐你将demo.com重定向至www.demo.com，点ADD即可，然后他会提示你添加两条解析记录，一个是@开头的和CNAME开头的，添加二级域名记录的方法与一级域名一致。

  2. 此处以阿里云域名为例进行说明，购买域名后，进行实名认证，然后进入阿里云域名控制台，点云解析进去，找到你刚买的域名，点进去添加Vercel所需的解析记录，注意博主这里是一级域名，需要解析两个，一个是@，另外一个是www，Vercel建议是@的解析重定向至www。但是如果是解析二级域名只需要解析一条，一级域名解析如图示操作即可。
  3. 回到Vercel刚刚查看域名的地方，如果操作没问题，应该会显示域名配置成功的提示，此时就可以通过自定义域名来访问我们搭建的网站了。
  4. 当你有了新的域名之后，需要[BlogRoot]_config.yml文件中的url配置项为自己的新域名，这样博客的文章链接才会正确生成。
